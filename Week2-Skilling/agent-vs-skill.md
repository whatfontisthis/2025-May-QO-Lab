# 언제 서브에이저트를 쓰고 언제 스킬을 쓰나요?
한줄 요약: **스킬은 반복 업무용, 서브에이전트는 조사·분석용**으로 보면 됩니다. 비즈니스에서는 보통 **스킬로 표준화하고, 서브에이전트로 복잡한 일을 처리**하는 조합이 가장 실용적입니다.

## 한 줄 기준
1. **스킬**: 늘 같은 형식으로 처리하는 일.
2. **서브에이전트**: 조사, 분석, 비교처럼 생각을 많이 해야 하는 일.
3. 둘 다 필요할 때는 **스킬이 절차를 정하고, 서브에이전트가 실행**합니다.

## 스킬 vs 서브에이전트 비교표

| 구분 | 스킬 | 서브에이전트 |
|------|------|------------|
| **핵심 역할** | 반복되는 프로세스 표준화 | 복잡한 작업 전문가 |
| **결과물 특성** | 형식 통일 (문서·이메일) | 조사·분석·판단 |
| **명령어** | `/skill [인자]` | `@agent 작업 설명` |
| **호출 방식** | 명시적 (사용자 직접 호출) | 자동화 가능 (메인 에이전트가 필요시 호출) |
| **맥락 길이** | 짧음 (인자만 필요) | 길어도 괜찮음 (독립 컨텍스트) |
| **중간 판단** | 거의 없음 (정해진 절차만) | 많음 (찾고·검증하고·판단) |
| **사용 예시** | 회의록 정리 · 제안서 초안 · 보고서 형식 통일 | 시장 조사 · 경쟁사 분석 · 계정별 맞춤 리서치 |

## 실습 과제별 사용 기준

### ✅ 스킬을 쓰는 경우
- 회의록 → 3줄 요약 + 액션 아이템
- 고객 메일 → 응대 이메일 템플릿
- 데이터 → 월간 보고서 형식 고정
- 채용 공고 → 우리 회사 톤으로 정규화
- 반복되는 문서 → 항상 같은 형식으로 처리

### ✅ 서브에이전트를 쓰는 경우
- 경쟁사 3곳 조사 → 비교표 + 인사이트 도출
- 시장 트렌드 + 우리 강점 분석 → 전략안 작성
- 고객 피드백 + 제품 로드맵 → 우선순위 제안
- 여러 파일 검토 → 일관성 체크 + 개선안 도출
- 불확실한 정보 → 찾고 검증해서 최종 결론 제시

## 비즈니스 워크플로 예시
### 1. 리드 발굴
- **스킬**: 리드 정보 입력 양식, 평가 기준, 결과 포맷 통일.
- **서브에이전트**: 회사 조사, 산업 확인, 의사결정자 탐색.

### 2. 제안서 작성
- **스킬**: 제안서 구조, 톤, 가격표 형식 고정.
- **서브에이전트**: 고객 배경 조사, 과제 정리, 맞춤 메시지 생성.

### 3. 고객 미팅 후속 처리
- **스킬**: 회의록, 액션 아이템, 팔로업 메일 템플릿.
- **서브에이전트**: 미팅 내용에서 리스크, 우선순위, 추가 질문 추출.

### 4. 경쟁사 분석
- **스킬**: 비교표 포맷, 평가 항목, 결과 정리 방식.
- **서브에이전트**: 경쟁사 자료 수집, 기능 비교, 차별점 요약.

## 실무용 규칙
- **반복 가능하면 스킬**
- **조사·판단·검증이 많으면 서브에이전트**
- **둘 다 필요하면 스킬 + 서브에이전트 조합**

## What is
- **스킬**: 특정 업무를 정해진 방식으로 처리하게 하는 재사용 규칙.
- **서브에이전트**: 특정 업무를 대신 맡는 작은 전문 작업자.
- **SOP [standard operating procedure]**: 업무를 항상 같은 방식으로 처리하는 표준 절차.
